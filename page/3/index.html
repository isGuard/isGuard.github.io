redis安装在liunx

发表于 2019-04-21 分类于 redis
本文字数： 532 阅读时长 ≈ 1 分钟

下载包
https://redis.io/
如包名为redis-5.0.4.tar.gz

上传至服务器：如/tmp
移动到目录下
cd /tmp

  1. 解压
    tar -zvxf redis-5.0.4.tar.gz
  2. 移动到/usr/local并重名名为redis
    mv redis-5.0.4 /usr/local/redis
  3. 切换目录
    cd /usr/local/reids/
  4. 编译
    make
  5. 添加启动目录
    make install
  6. 测试访问
    redis-s > tab >显示 redis-server 即可
  7. 打开配置文件
    vi redis.conf
  8. 更改 完成后
    :wq
  9. 启动服务
    redis-server ./redis.conf
    进程查看ps -ef | grep redis
  10. 访问服务
    redis-cli
  11. 退出
    exit
  12. 关闭
  13. 配置成服务 检测
    chkconfig --list | grep redis

密文搜索

发表于 2019-04-06 分类于 算法
本文字数： 1.4k 阅读时长 ≈ 1 分钟

福尔摩斯从X星收到一份资料，全部是小写字母组成。
他的助手提供了另一份资料：许多长度为8的密码列表。
福尔摩斯发现，这些密码是被打乱后隐藏在先前那份资料中的。

请你编写一个程序，从第一份资料中搜索可能隐藏密码的位置。要考虑密码的所有排列可能性。

数据格式：

输入第一行：一个字符串s，全部由小写字母组成，长度小于1024*1024
紧接着一行是一个整数n,表示以下有n行密码，1<=n<=1000
紧接着是n行字符串，都是小写字母组成，长度都为8

要求输出：
一个整数, 表示每行密码的所有排列在s中匹配次数的总和。

例如：
用户输入：
aaaabbbbaabbcccc
2
aaaabbbb
abcabccc

则程序应该输出：
4

这是因为：第一个密码匹配了3次，第二个密码匹配了1次，一共4次。

资源约定：
峰值内存消耗（含虚拟机） < 512M
CPU消耗 < 5000ms

请严格按要求输出，不要画蛇添足地打印类似：“请您输入…” 的多余内容。

本文字数： 844 阅读时长 ≈ 1 分钟

高并发处理 : 在当前进程取出对象，不会冲突
放入的东西
各个线程保存的东西是分开只与进程有关系
每个进程会处理自己的内容

ThreadLocal

ThreadLocal的是一个本地线程副本变量工具类。

主要用于将私有线程和该线程存放的副本对象做一个映射，各个线程之间的变量互不干扰，在高并发场景下，可以实现无状态的调用，特别适用于各个线程依赖不通的变量值完成操作的场景。
